Módulo: Sistema Operacional Linux

História, Filosofia e Arquitetura de Sistemas

1. A História e a Filosofia do Linux

Para entender o Linux, precisamos voltar ao Unix, um sistema operacional multifuncional criado nos anos 1970 nos laboratórios AT&T Bell. O Unix introduziu conceitos de design que moldaram a computação moderna.

O Manifesto GNU e o Software Livre

Em 1983, Richard Stallman iniciou o Projeto GNU com o objetivo de criar um sistema operacional totalmente livre, compatível com o Unix. Embora tenham criado quase todas as ferramentas de software necessárias (compiladores, editores de texto, shells), ainda faltava o "coração" do sistema: o Kernel.

O Surgimento do Linux

Em 1991, um estudante finlandês chamado Linus Torvalds começou a desenvolver um núcleo de sistema operacional por hobby, frustrado com as limitações do Minix (um sistema educacional). Ele combinou seu núcleo (o Linux) com as ferramentas do projeto GNU. Assim nasceu o sistema que hoje conhecemos formalmente como GNU/Linux.

Filosofia Unix aplicada ao Linux:
  • Escreva programas que façam apenas uma coisa, mas que a façam bem feita.
  • Escreva programas que trabalhem juntos (conceito de Pipes).
  • Escreva programas para manipular fluxos de texto, pois essa é uma interface universal.

2. Arquitetura do Sistema

A arquitetura do Linux é baseada na divisão clara de privilégios de execução no hardware (Modo Usuário vs. Modo Kernel), garantindo estabilidade e segurança.

Camadas do Sistema

  • Hardware: A infraestrutura física (Processador, Memória, Discos, Placas de Rede).
  • Kernel (Núcleo): O software que se comunica diretamente com o hardware. Ele gerencia o tempo de CPU, a alocação de memória e o acesso aos dispositivos. O Linux é um Kernel Monolítico, o que significa que os drivers de dispositivos rodam no mesmo espaço de memória do núcleo principal para maior desempenho.
  • Shell (Interpretador de Comandos): A interface de texto que traduz os comandos digitados pelo usuário para que o Kernel os execute.
  • Aplicações de Usuário: Editores, navegadores, servidores web (Apache, Nginx) e ferramentas de sistema que rodam no User Space.

Distribuições Linux (Distros)

Como o Kernel Linux é de código aberto, empresas e comunidades criam pacotes completos (Kernel + Utilitários + Gerenciador de Pacotes). Chamamos isso de Distribuições. Exemplos:

  • Foco em Servidores/Estabilidade: Debian, Red Hat Enterprise Linux (RHEL), Ubuntu Server, Rocky Linux.
  • Foco em Usuário Final: Ubuntu Desktop, Linux Mint, Fedora.
  • Foco em Segurança/Pentest: Kali Linux.
