Linux e a Organização do Hardware — Gerenciamento de Memória, Paginação e Processos

1. O Casamento entre o Kernel e a MMU

O Linux interage diretamente com a MMU (Memory Management Unit) do processador para implementar o conceito de Memória Virtual. Isso isola os processos, fazendo com que cada programa "pense" que tem acesso exclusivo a todo o mapa de endereçamento de memória do sistema.

Paginação (Paging)

A memória física (RAM) e a memória virtual são divididas em blocos de tamanho fixo chamados Pages (Páginas), tipicamente de 4 KB na arquitetura x86/x64. O Kernel gerencia as tabelas de páginas (Page Tables) para traduzir endereços virtuais em endereços físicos de hardware.

2. Memória Swap e o Algoritmo de Substituição

Quando a memória RAM física fica cheia, o Linux utiliza uma área de armazenamento secundário (Disco Rígido ou SSD) chamada Espaço de Swap para estender a memória volátil.

Como funciona o Swapping?

O Kernel monitora as páginas de memória que não estão sendo utilizadas ativamente. Através de algoritmos baseados em LRU (Least Recently Used), ele move essas páginas inativas para a Swap (Page Out), liberando a RAM para os processos prioritários. Se o processo precisar daquele dado novamente, ocorre uma interrupção por hardware chamada Page Fault, fazendo o Kernel trazer o dado de volta para a RAM (Page In).

Consequência de Hardware (Thrashing): Se o sistema passar mais tempo movendo páginas de/para a Swap do que executando instruções úteis na CPU, ocorre o fenômeno de Thrashing, reduzindo drasticamente a performance do computador.

3. Prática: Monitorando o Hardware pelo Terminal

Podemos observar o gerenciamento de memória e da CPU diretamente através do sistema de arquivos virtual /proc ou por utilitários nativos.

Verificando o consumo de RAM e Swap:

# Exibe a memória total, usada e livre (físico e swap) em Megabytes
free -m

# Lendo dados diretamente do Kernel através do sistema virtual
cat /proc/meminfo

Estatísticas de Paginação Virtual:

# Atualiza a cada 2 segundos as estatísticas de memória virtual e I/O de swap (si = swap in, so = swap out)
vmstat 2

4. Gerenciamento de Processos e Escalonamento

Para a CPU, um processo é uma sequência de instruções em execução. O Linux utiliza estruturas chamadas task_struct para representar cada processo no sistema.

Estados de um Processo:

  • Running (R): Executando na CPU ou pronto para executar.
  • Sleeping (S/D): Aguardando algum evento de hardware (ex: leitura de disco).
  • Zombie (Z): Terminou a execução, mas ainda possui registro na tabela do Kernel aguardando a leitura do processo pai.

Comandos de Monitoramento de Processos:

# Gerenciador de tarefas em tempo real (interativo)
top

# Lista todos os processos ativos no hardware com detalhes de dono e uso de CPU
ps aux
