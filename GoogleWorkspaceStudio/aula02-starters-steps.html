Starters: as três famílias de gatilhos

O Starter é o evento que dispara o fluxo. Todos os gatilhos do Studio caem em uma destas três categorias:

Comunicação

Eventos de mensagens: chegou um novo e-mail (com um filtro/rótulo), uma mensagem no Chat, etc.

Mudança de dados

Alterações de estado: uma nova linha em uma Planilha, um formulário respondido, um arquivo novo.

Temporal / Manual

Agendado (todo dia às 8h) ou manual (você clica para rodar / roda a partir do Gmail).

Escolher o Starter certo é a decisão de design mais importante do fluxo: ele define quando e com que dados tudo começa. Um Starter mal escolhido gera fluxos que disparam demais (ruído) ou de menos (perdem eventos).

Escolhendo e configurando o Starter

StarterDispara quando…Bom para
E-mail recebidoChega e-mail que casa com um filtro (remetente, rótulo, palavra)Triagem, salvar anexos, respostas
Nova linha na PlanilhaUma linha é adicionada a uma abaProcessar cadastros, pedidos, respostas de Forms
AgendadoEm um horário/frequência definidosRelatórios e resumos periódicos
ManualVocê aciona sob demandaTarefas pontuais, testes, "botão" de rotina

Boa prática: comece com o gatilho mais específico possível. Em "e-mail recebido", use filtros (de um remetente, com um rótulo) para o fluxo não disparar a cada mensagem da caixa.

Nem todo evento imaginável existe como Starter, e a lista evolve. Se o gatilho que você quer não existe, às vezes dá para simular com um agendamento que verifica a condição periodicamente.

Steps dos apps Google (ações)

Os Steps executam ações. Os mais comuns operam sobre os próprios apps do Workspace:

AppAções típicas
GmailEnviar e-mail, criar rascunho de resposta, aplicar rótulo, marcar com estrela, arquivar
PlanilhasAdicionar linha, atualizar uma linha/célula existente
DriveCriar pasta, salvar anexo, mover/copiar arquivo
DocsCriar documento, anexar conteúdo a um documento
AgendaCriar evento, ler próximos eventos
ChatEnviar mensagem para um espaço ou pessoa
TasksCriar tarefa

Repare no par ler × escrever: alguns passos obtêm dados (ler próximos eventos) e outros alteram dados (adicionar linha). Combinar leitura + IA + escrita é a receita da maioria dos fluxos úteis.

Variáveis: a cola entre os passos

Cada passo produz uma saída (output). Essa saída fica disponível como variável para os passos seguintes usarem como entrada (input).

E-mail recebido
saída: assunto, corpo, anexos
Resumir
entrada: corpo → saída: resumo
Enviar no Chat
entrada: resumo

Na prática, ao configurar um passo você verá um seletor para inserir variáveis dos passos anteriores (por exemplo, "corpo do e-mail", "nome do remetente", "resumo gerado"). É assim que os dados fluem pela esteira.

Modelo mental: pense em cada passo como uma função — recebe entradas, devolve saídas. Encadear passos é compor funções. Quem já programou reconhece o padrão de pipeline.

Os passos de IA: onde o Gemini pensa

Além das ações mecânicas, o Studio traz passos que usam o Gemini para interpretar, gerar e decidir. São eles que transformam um fluxo em um agente:

Passo de IAO que faz
Ask Gemini (Perguntar ao Gemini)Uso geral: gerar texto, reescrever, responder, analisar
Summarize / Recap (Resumir)Condensar um conteúdo longo em pontos-chave
Extract (Extrair)Puxar dados estruturados segundo um schema que você define
Check If (Verificar se)Condição com lógica E/OU — ramifica o fluxo
Decide (Decidir)Pede ao Gemini um julgamento entre opções
Filter (Filtrar)Percorre uma lista e mantém itens que atendem a um critério

Guarde a distinção: Ask/Summarize geram conteúdo; Extract estrutura dados; Check If / Decide / Filter controlam o fluxo com base em julgamento semântico. Vamos aprofundar cada um na Aula 3.

Lógica com IA: Check If e Decide

Fluxos raramente são uma linha reta. Muitas vezes precisamos de "só faça isto se…". É aí que entram os passos de condição:

Check If

Avalia uma condição (com E/OU) e ramifica: se verdadeiro, segue por um caminho; senão, por outro. Ex.: "se o e-mail contém uma fatura E é de um fornecedor conhecido".

Decide

Delega o julgamento ao Gemini: "classifique a prioridade deste e-mail como Alta, Média ou Baixa". A saída (a categoria) orienta os próximos passos.

Cuidado com decisões da IA. Um julgamento do Gemini é uma estimativa, não uma certeza. Para ações irreversíveis (excluir, enviar a clientes), prefira que a IA sugira e um humano confirme.

Laboratório 1 — Triagem de e-mails prioritários

Vamos montar um agente que identifica e-mails importantes e avisa você, sem mexer no que não deve.

E-mail recebido
Decide
prioridade?
Aplicar rótulo
Notificar no Chat
  1. Starter: "E-mail recebido" (opcionalmente filtrado por caixa de entrada principal).
  2. Decide: peça ao Gemini para classificar a prioridade em Alta / Média / Baixa, explicando o critério (ex.: pedidos diretos, prazos, remetentes-chave).
  3. Check If: continue apenas quando a prioridade for Alta.
  4. Ação — rótulo: aplique o rótulo "Prioritário" no e-mail.
  5. Ação — Chat: envie uma mensagem: "📌 E-mail prioritário de {remetente}: {assunto}".
  6. Teste com um e-mail real de exemplo e ajuste o critério do Decide conforme o resultado.

Esse fluxo informa e organiza, mas não apaga nem responde nada — é seguro para produção.

Laboratório 2 — Capturar anexos e registrar no Sheets

Um clássico do escritório: quando chega um e-mail com anexo, guardar o arquivo no Drive e registrar os dados numa planilha — automaticamente.

E-mail c/ anexo
Salvar anexo no Drive
Extract
dados do e-mail
Adicionar linha no Sheets
  1. Starter: "E-mail recebido" com filtro para mensagens que têm anexo (ex.: rótulo "Notas").
  2. Ação — Drive: salve o anexo numa pasta específica (lembre-se: escrever em Drive compartilhado pode ter restrições; use uma pasta sua).
  3. Extract: defina um schema — por exemplo { remetente, data, valor, descrição } — e o Gemini extrai esses campos do e-mail.
  4. Ação — Sheets: adicione uma linha com os campos extraídos + o link do arquivo salvo.
  5. Teste e verifique se os campos vieram corretos; refine o schema e a instrução.

Este é o padrão ler → estruturar (IA) → escrever, um dos mais úteis de todos. Você o reaproveitará em dezenas de cenários.

Boas práticas ao montar flows

  • Um objetivo por fluxo. Fluxos enxutos são fáceis de entender, testar e corrigir.
  • Nomeie com clareza. "Triagem — e-mails prioritários" é melhor que "Fluxo 3".
  • Teste com dados reais antes de ativar; comece por ações que não alteram nada.
  • Trate o caminho de erro. O que acontece se o e-mail não tiver anexo? Se a IA não achar o valor?
  • Documente a intenção em uma descrição — seu "eu do futuro" agradece.
  • Respeite os limites (nº de passos): se estourar, quebre em dois fluxos encadeados.

Pense como engenheiro de software: entrada → processamento → saída, com casos de borda previstos. Um bom flow é um pequeno sistema — merece o mesmo cuidado de design.

Exercícios da Aula 2

1. Para cada tarefa, indique o Starter ideal e justifique: (a) avisar a equipe quando um formulário for respondido; (b) enviar um resumo semanal toda sexta às 17h; (c) salvar toda nota fiscal que chega por e-mail.

2. Reproduza o Laboratório 1 (triagem de e-mails). Ajuste o critério do Decide e documente como a classificação mudou. Entregue prints do fluxo e de uma execução.

3. Reproduza o Laboratório 2 (anexos → Sheets). Projete o schema do Extract para o seu caso e explique cada campo escolhido.

4. Desafio. Desenhe (em papel ou diagrama) um fluxo com pelo menos um Check If que trate um caminho de erro (ex.: "se não houver anexo, apenas registre um aviso"). Identifique variáveis de entrada e saída de cada passo.

Resumo da Aula 2

Os Starters se dividem em três famílias — comunicação, mudança de dados e temporal/manual — e definem quando o fluxo começa. Os Steps executam ações nos apps Google (Gmail, Sheets, Drive, Docs, Agenda, Chat, Tasks), e as variáveis transportam dados de um passo ao seguinte, formando uma esteira. Os passos de IA — Ask Gemini, Summarize, Extract, Check If, Decide e Filter — acrescentam interpretação, estruturação e decisão. Você construiu dois agentes completos: triagem de e-mails e captura de anexos. Na Aula 3, mergulhamos fundo na IA: Gems, prompts, extração estruturada e casos avançados.
