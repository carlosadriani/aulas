Aula 8: Extração de Relatórios com a Função QUERY

Na aula anterior, conhecemos a função FILTER, que nos permite extrair linhas de uma tabela de forma dinâmica. Contudo, e se você precisasse de extrair apenas algumas colunas específicas, ordenando os nomes em ordem alfabética e trazendo somente quem atingiu uma determinada pontuação? Fazer isso juntando várias fórmulas comuns deixaria a planilha lenta. Para resolver isso, o ecossistema Google possui a função QUERY, capaz de ler instruções escritas em formato de frase estruturada para montar relatórios imediatos.

Objetivo da Aula: Compreender a estrutura de escrita da função QUERY, dominar as três cláusulas essenciais de seleção administrativa (SELECT, WHERE e ORDER BY) e construir tabelas de auditoria cruzada automáticas.

1. Como Funciona a Sintaxe da QUERY?

Diferente de outras funções, a QUERY usa instruções em inglês que funcionam como ordens diretas para o sistema. A estrutura básica da fórmula possui três argumentos:

=QUERY(intervalo_dados; "frase_de_comando"; [linhas_de_cabeçalho])
  • intervalo_dados: A matriz ou tabela completa de onde os dados serão extraídos (Ex: A1:E500).
  • "frase_de_comando": O texto escrito entre aspas duplas contendo as instruções do que fazer com as informações.
  • linhas_de_cabeçalho: Número que indica quantas linhas de títulos a sua tabela possui no topo (geralmente usamos o número 1).

2. As Três Cláusulas Principais (O Vocabulário)

Para conversar com a função QUERY, montamos a frase utilizando três palavras-chave principais, sempre escritas em letras maiúsculas dentro das aspas:

Cláusula O que ela faz Tradução Prática para a Rotina
SELECT Indica quais colunas devem ser trazidas para o relatório final. "Traga apenas as colunas A e C, pule o resto."
WHERE Aplica as regras e condições de filtragem. "Apenas mostre quem é da Secretaria e tem mais de 30 dias de prazo."
ORDER BY Organiza a classificação de exibição dos dados. "Ordene o resultado final do maior valor para o menor."

3. Cenário Prático: Construindo um Painel de Auditoria

Imagine que o setor de Recursos Humanos da Universidade possui uma planilha principal chamada CadastroDocentes com as seguintes colunas:

  • Coluna A: Nome do Professor
  • Coluna B: Faculdade/Centro
  • Coluna C: Regime de Trabalho (Integral ou Parcial)
  • Coluna D: Quantidade de Turmas Alocadas

O diretor precisa de um relatório urgente que exiba **apenas o Nome (A) e o Regime de Trabalho (C)**, mas apenas dos professores lotados na faculdade de **"Tecnologia"** e organizados por **ordem alfabética (A)**.

A Solução com uma única Fórmula QUERY:
Você abre uma aba em branco e, na célula A1, digita apenas esta fórmula:
=QUERY(CadastroDocentes!A1:D500; "SELECT A, C WHERE B = 'Tecnologia' ORDER BY A ASC"; 1)
Análise da Instrução Interna:
  • SELECT A, C: Diz à planilha para ocultar as colunas B e D no relatório final.
  • WHERE B = 'Tecnologia': Filtra a tabela original. Note que o termo textual procurado deve ficar entre aspas simples ('Tecnologia').
  • ORDER BY A ASC: Organiza a listagem de nomes (Coluna A) em ordem crescente (ASC), ou seja, de A a Z.

4. Cuidados Especiais e Regras de Ouro

  • Letras das Colunas: Dentro das aspas da QUERY, as colunas devem ser escritas obrigatoriamente com letras **maiusculas** (Use SELECT A e nunca SELECT a).
  • Aspas Simples para Texto: Como a frase de comando já fica dentro de aspas duplas, qualquer texto interno de comparação no filtro precisa usar aspas simples (Ex: WHERE C = 'Integral').
  • Números Sem Aspas: Se você for filtrar um valor numérico, ele entra direto sem aspas (Ex: para filtrar quem tem mais de 3 turmas, use WHERE D > 3).
Exemplo Combinado (Texto + Número):
Caso precise de puxar professores em regime 'Integral' que tenham mais de 4 turmas alocadas: =QUERY(A1:D500; "SELECT A WHERE C = 'Integral' AND D > 4"; 1)
