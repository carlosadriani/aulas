Aula 1: Interface e Ciclo de Vida do Dado

Nesta primeira aula do curso de Planilhas Google, compreenderemos como a aplicação estrutura as informações na memória visual e de que forma o ciclo de vida do dado (da captura ao tipo de armazenamento) impacta na construção de relatórios eficientes para a área de TI.

Objetivo da Aula: Dominar a navegação estrutural do ambiente, diferenciar os tipos primitivos de dados e executar importações limpas a partir de origens externas.

1. A Anatomia da Interface (Matriz de Dados)

Diferente de um editor de texto, a planilha trabalha sob o conceito clássico de matriz bidimensional. Cada elemento possui um endereço único e imutável:

  • Colunas: Indicadas por letras (Eixo X). Representam os atributos/campos no nosso banco de dados.
  • Linhas: Indicadas por números (Eixo Y). Representam as tuplas/registros.
  • Célula: A interseção exata de uma coluna com uma linha (Ex: B5).
  • Intervalo (Range): Um bloco de dados selecionado de forma contínua, referenciado pelo uso de dois pontos :.
Exemplo Prático de Endereçamento:
Anotar o intervalo A1:C10 significa dizer que estamos manipulando os dados da coluna A até a coluna C, partindo da linha 1 até a linha 10. É a estrutura ideal para criarmos uma tabela de inventário ou logs rápidos.

2. Ciclo de Vida e Tipagem de Dados

Um erro comum no desenvolvimento de planilhas é ignorar a tipagem do dado. O Planilhas Google reconhece automaticamente quatro tipos primitivos fundamentais de dados. Alinhá-los corretamente evita falhas lógicas em cálculos e scripts:

Tipo de Dado Alinhamento Padrão Exemplo em TI Comportamento Esperado
Texto / String Esquerda "Servidor_01" Não realiza operações matemáticas nativas.
Numérico Direita 192 ou 45.50 Aceita operadores aritméticos e funções de agregação.
Data / Time Direita 15/06/2026 Armazenado internamente como número de série (permite somar dias).
Booleano Centralizado VERDADEIRO Usado para flags, validações de segurança e lógica condicional.
Dica do Prof: Se você digitou um número e ele ficou alinhado à esquerda por padrão, a planilha o interpretou como texto! Isso quebrará suas futuras fórmulas.

3. Importação de Arquivos (CSV vs XLSX)

Em muitos casos, não digitaremos dados manualmente célula por célula. O ecossistema nos permite herdar dados de sistemas externos:

  1. Mecanismo de Importação: Acesse Arquivo > Importar > Fazer Upload.
  2. Arquivos .CSV (Comma-Separated Values): O formato universal de exportação de bancos de dados textuais. O separador padrão pode ser a vírgula ou o ponto e vírgula.
  3. Arquivos .XLSX: O formato proprietário do Microsoft Excel. O Planilhas Google faz a conversão direta mantendo a maioria das funções.
